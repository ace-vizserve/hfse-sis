## Project layout

Single deployable at the repo root. App Router lives in `app/`; there is no wrapper subdirectory.

```
proxy.ts                         auth + role gate
app/(auth)/login/
app/(dashboard)/                 neutral shell: /, /account, redirect stubs
app/(markbook)/markbook/         grading, report-cards, sections, change-requests, audit-log
app/(attendance)/attendance/     dashboard (analytics) + sections/ (picker, KD #55) + [sectionId] (daily writer, KD #47)
app/(evaluation)/evaluation/     form-class-adviser writeups (KD #49)
app/(admissions)/admissions/     pre-enrolment funnel (KD #51) + applications/{,[enroleeNumber],closed} (closed = terminal-stage archive, KD #70) + cohorts/{stp,medical,promised} (lifecycle observation lenses; pass-expiry mirror moved to /records-only since pre-enrolment travel-doc lapses surface via ?status=expired per KD #70)
app/(attendance)/attendance/     dashboard + sections/ + [sectionId] (daily writer, KD #47/#55) + students/[studentNumber] (per-student attendance detail)
app/(parent)/parent/             dedicated parent surface — sidebar removed; thin top header (school identity + back-to-portal button); /parent dashboard lists children with active publications, /parent/report-cards/[studentId] shows the card or a case-aware empty state (revoked / expired / scheduled / not-enrolled). Parent auth lives on the parent_session HMAC cookie (KD #65); the layout never bounces parents to / or /login — anonymous + null-role JWTs route to NEXT_PUBLIC_PARENT_PORTAL_URL
app/(p-files)/p-files/           documents repository — page-level role split (officer = p-file/superadmin sees chase strip + chase priority; admin/school_admin sees read-only oversight without those operational tiles)
app/(records)/records/           enrolled students (cross-year via studentNumber) + dashboard (registrar = operational, school_admin/admin/superadmin = oversight) + audit + cohorts/{stp,medical,pass-expiry}
app/(sis)/sis/                   admin hub: ay-setup, calendar (audience-filtered, primary/secondary/all per KD #76), sections, sync-students, admin/{approvers,discount-codes,evaluation-checklists,school-config,settings,subjects,users}
app/api/                         one folder per resource
lib/                             auth, compute, audit, schemas, <module>/{queries,mutations,dashboard,drill}.ts; lib/sis/{environment,seeder}/* for the test-AY flow (KD #52); lib/sis/cohorts.ts (cross-module STP / medical / pass-expiry lifecycle observation queries, served by /api/sis/cohorts/[cohort]); lib/sis/process.ts (lifecycle observability — composite-timeline rollups, served by /api/sis/drill/[target]); lib/sis/levels.ts (LEVEL_LABELS / LEVEL_CODES / compareLevelLabels / canonicalizeLevelLabel — single source of truth for word-form level labels per migration 029); lib/sis/terms.ts (getTermForDate / preloadTermsForAYs / termForDateInPreloaded — shared date→term resolution per KD #67 + #68); lib/sis/section-transfer.ts (transferStudentSection — atomic withdraw+insert for mid-year section moves per KD #67); lib/sis/section-history.ts (getSectionTransfersForStudent — audit-log-derived cross-AY transfer history for the Records detail page); lib/dashboard/* shared dashboard primitives — range.ts (preset resolution + delta math), windows.ts (term/AY windows, uses service client per KD #54), insights.ts (per-module narrative engines), priority.ts (PriorityPayload type for operational top-of-fold per KD #57); lib/<module>/drill.ts owns DrillRow shape(s) + buildDrillRows + applyTargetFilter + defaultColumnsForTarget per KD #56; lib/markbook/change-request-status.ts (shared status pill config consumed by both admin queue + teacher view); lib/sidebar/{registry,use-realtime-badges}.ts (shared module-sidebar config + generalized live-badge hook per KD #58); lib/schemas/subject.ts (SubjectCreateSchema for catalog INSERT per KD #72); lib/auth/staff-list.ts (getTeacherList — auth users with role='teacher' for staff/teacher pickers; sibling of lib/auth/teacher-emails.ts; 5-min cache shared with the 'teacher-emails' invalidation tag); lib/p-files/freshen-document-statuses.ts (relocated from lib/sis/ — bidirectional auto-flip Valid+past-expiry → Expired AND Expired+future-expiry → Valid per KD #60; mounted at the top of every page RSC that reads document status)
components/                      ui (shadcn, including new `command.tsx` cmdk wrapper) + per-module folders + shared `module-sidebar.tsx` + `module-sidebar/{sidebar-header,sidebar-nav-item,sidebar-profile,sidebar-quick-action}.tsx` (KD #58 — replaces the 8 per-module `<module>-sidebar.tsx` files + the topbar `<ModuleSwitcher>`) + `topbar-module-switcher.tsx` (used only on the neutral `/` + `/account` shells, where there is no module sidebar to host the switcher); components/sis/command-palette.tsx (global Cmd+K palette — navigation + student search; trigger lives in module-sidebar header + topbar); components/sis/cohorts/{cohort-page-shell,medical-cohort-table,pass-expiry-cohort-table,stp-cohort-table}.tsx (cross-module cohort tables consumed by both /admissions/cohorts and /records/cohorts); components/sis/{lifecycle-aggregate-card,lifecycle-aggregate-row,student-lifecycle-timeline,stp-application-card,residence-history-editor}.tsx (Sprint 27 lifecycle + STP surfaces); components/sis/{section-transfer-dialog,section-roster-table}.tsx (KD #67 — section roster + Move action); components/admissions/completeness-table.tsx (Phase 2A admissions chase table per KD #70); components/dashboard/* shared dashboard UI — DashboardHero, ComparisonToolbar, MetricCard (with `drillSheet` slot), InsightsPanel, ActionList, PriorityPanel (operational top-of-fold per KD #57), DrillDownSheet (universal toolkit), ChartLegendChip (gradient pill per `09a-design-patterns.md` §10), charts/{trend,comparison-bar,donut,sparkline}-chart.tsx; components/<module>/drills/{<module>-drill-sheet,chart-drill-cards}.tsx for drill wrappers
```
